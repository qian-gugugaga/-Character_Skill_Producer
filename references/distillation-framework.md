# 行为蒸馏方法论

> 从角色信息到可运行的行为程序，核心方法论

## 一、最高质量模式

CSP 默认采用 Highest Fidelity Mode。目标不是省 token，而是生成资料完整、证据可追溯、行为可执行、资料日期清晰的角色 Skill。

最低要求：

- 优先使用本地脚本检索核心站点。
- 每条来源记录检索日期。
- 重要结论至少两个独立来源。
- 核心行为模式必须有跨场景证据。
- 跨媒体角色必须记录媒体覆盖和未覆盖内容。
- 最终 Skill 必须写明资料更新日期和更新响应规则。

---

## 二、行为模式的识别标准

一个观察要被认定为「核心行为模式」而非「偶尔一次的行为」，必须通过二重验证。

### 验证 1：跨场景复现

同一行为模式出现在角色经历的至少 2 个不同场景。

例：高松灯

- 场景 A：爱音邀请她组乐队 → 沉默很久后说「一辈子」
- 场景 B：在天台追回爱音 → 气喘吁吁说「都是灯的错」

结论：在人际关系关键时刻，用极其笨拙但极端真挚的方式表达需求。这可以成为核心行为模式。

### 验证 2：可执行性

能据此推断角色对新情境的可能反应。

例：如果灯被一个新朋友邀请去参加活动：

- 她会先犹豫。
- 如果对方表现出真正的需要，她会答应，但可能提出一个过重的承诺。

这能推断新反应，可以写入 Skill。

**如果只通过 1 重**：降级为低置信度行为，仍可纳入但不能写成核心。  
**如果 0 重**：可能只是特定场景的一次性反应，不纳入。

## 三、表达质感的量化方法

### 3.1 句式指纹

从角色台词中抽取样本，统计：

| 维度 | 测量方式 |
|---|---|
| 平均句长 | 字数 / 句数 |
| 疑问句比例 | 疑问句数 / 总句数 |
| 句中停顿频率 | 省略号、停顿、短句中断出现频率 |
| 感叹句比例 | 感叹句 / 总句数 |
| 敬语密度 | 敬语标记 / 句数 |

### 3.2 风格坐标

```text
话多 ←→ 话少
长句 ←→ 短句/碎片
直接 ←→ 迂回
热情 ←→ 冷淡
敬语 ←→ 简体
稳定 ←→ 情绪起伏大
逻辑 ←→ 直觉/跳跃
```

### 3.3 语言标志（日文角色特化）

- 自称词：私 / 僕 / 俺 / あたし / わし / 自分 / うち
- 对称词：あなた / 君 / お前 / あんた / 名字+さん / 名字+くん
- 语尾：～ですわ / ～だよ / ～なのだ / ～にゃ / ～っす
- 口癖：重复出现的特定短语
- 禁忌词：角色不会用的表达方式

语言标志不是为了机械复读，而是为了让中文输出保留原角色的节奏、距离感和社会位置。

## 四、矛盾处理原则

矛盾是角色的核心特征，不是需要修复的 Bug。

### 三种矛盾类型

1. **情境性矛盾**
   - 对 A 类型的人温柔，对 B 类型的人冷酷。
   - 处理：分情境记录，不强求统一。

2. **发展性矛盾**
   - 早期反应 A，后期反应 B。
   - 处理：标注阶段，以当前 Skill 采用的时间线为准，同时记录演变。

3. **本质性张力**
   - 既渴望 X 又害怕 X。
   - 处理：明确记录为核心矛盾。

错误方式：

- 选一边忽略另一边。
- 编一个没有证据的调和解释。
- 假装矛盾不存在。

## 五、资料时间边界

角色作品会更新。游戏会出新版本，动画会有续作，官方访谈会补充设定。因此每个 Skill 必须记录资料时间边界。

必须写入：

- `manifest.json.research_completed_at`
- `manifest.json.latest_source_checked_at`
- `manifest.json.covered_until`
- `references/sources.json[*].retrieved_at`
- `SKILL.md` 的「资料时间边界」章节

当用户指出最新剧情、新版本、新活动或新设定与本 Skill 不一致时，不要硬拗。

标准回应：

```text
我的资料更新至 <latest_source_checked_at>，可能没有覆盖之后发布的内容。如果你有最新版 CSP，或可以提供新剧情 / 新资料链接，我可以帮你更新这个 Skill；这可能会消耗一些 Token。
```

用户提供的新材料是更新线索，不自动覆盖旧设定。必须验证来源和时间线。

## 六、跨媒体角色处理

跨媒体角色必须记录媒体覆盖，而不是只写一个泛化角色。

典型企划：

- BanG Dream!
- Love Live!
- Project Sekai
- 少女☆歌剧
- 其他动画、游戏、漫画、舞台并行展开的作品

### 媒体优先级

| 维度 | 优先来源 |
|---|---|
| 核心设定 | 官方角色介绍、主线动画、游戏主线、设定集 |
| 日常行为 | 活动剧情、卡面剧情、区域对话、短篇故事 |
| 表达质感 | 原作台词、语音、字幕、访谈中引用的原文 |
| 人际关系 | 主线冲突 + 活动日常互动 |
| 创作者意图 | 官方访谈，可辅助但不能替代角色实际行为 |

必须写 `references/research/06-media-coverage.md`，说明：

- 已覆盖媒体；
- 未覆盖媒体；
- 采用的时间线；
- 资料检索日期；
- 不同媒体冲突；
- 哪些结论只来自衍生材料。

---

## 七、信息不足时的处理

| 情况 | 处理 |
|---|---|
| 某个维度公开信息很少 | 标注信息不足，此维度为推测 |
| 只有粉丝/同人信息 | 降低置信度，标注为同人推测 |
| 信息相互矛盾且无法判断 | 并列呈现，让用户知道冲突存在 |
| 角色在作品中刻意保持神秘 | 保留神秘，注明原作未揭示 |
| 用户指出新剧情不同 | 承认资料日期边界，进入更新流程 |

---

## 八、角色 Skill vs 真人 Skill 的区别

| 维度 | 角色 Skill（CSP） | 真人思维 Skill |
|---|---|---|
| 核心 | 行为模式 + 表达质感 | 心智模型 + 决策启发式 |
| 动机 | 角色在故事中的驱动力 | 真人的价值观与目标 |
| 说话方式 | 模仿角色声音、节奏、口癖 | 模仿真人表达 DNA |
| 矛盾处理 | 保留角色内在冲突 | 保留真人观点演变与矛盾 |
| 验证方式 | 对比原作场景表现 | 对比真人已知立场 |
| 使用场景 | 角色扮演 / 对话 | 思维顾问 / 决策参考 |

## 九、质量自检清单

### 行为动态

- [ ] 每个核心行为模式都有至少 2 个不同场景的证据？
- [ ] 核心行为模式数在 3-7 个之间？
- [ ] 每个模式描述了触发条件？
- [ ] 每个模式说明了为什么这样做？
- [ ] 矛盾被保留而非消解？

### 表达质感

- [ ] 读起来有角色辨识度，不像通用 AI？
- [ ] 没有过度模仿成 caricature？
- [ ] 语言标志准确？
- [ ] 中文输出保留了原角色的距离感和节奏？

### 证据链

- [ ] 存在 `references/sources.json`？
- [ ] 每条来源有 `retrieved_at`？
- [ ] 黑名单来源没有被使用？
- [ ] 关键结论至少两个独立来源？
- [ ] 检索失败被记录？

### 时间边界

- [ ] `manifest.json` 有 `research_completed_at`？
- [ ] `manifest.json` 有 `latest_source_checked_at`？
- [ ] `manifest.json` 有 `covered_until`？
- [ ] `SKILL.md` 有资料时间边界？
- [ ] 用户指出新剧情时有更新回应规则？

### 整体

- [ ] 用这个角色的眼睛看一个新情境，反应合理可信？
- [ ] 不是原作台词拼凑，而是行为模式运转？
- [ ] 删掉名字和作品名后，还能认出这是哪个角色？
